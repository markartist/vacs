# Contributing

## Branch Posture
- `main`: protected baseline
- `feature/<topic>`: feature work
- `fix/<topic>`: targeted fixes
- `chore/<topic>`: maintenance

## Commit Guidance
- Keep changes scoped by concern (runtime, docs, governance, scripts)
- Use clear messages and include rationale for architectural changes

## Current Rule
Until planning is finalized, avoid introducing irreversible architecture coupling.
