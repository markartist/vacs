# Content Blueprint Module Plan (Draft)

Purpose: define scaffold-level module planning for a blueprint-first workflow model.

Status: draft and pending artifact review.

## Planned Module Boundaries
- Content Blueprint:
  - central strategic record
  - evidence intake references
  - strategic decision registry
  - section map for derivative outputs
- Derivative Workflow Modules (starting with Blog Workflow):
  - consume blueprint sections
  - apply channel-specific structure and packaging
  - retain links back to blueprint evidence/decisions

## Separation Requirements
- Evidence inputs are factual/source-linked records.
- Strategic decisions are intentional choices linked to evidence.
- Generated assets are derivative packages linked to blueprint records.

## Section-Based Output Requirement
Derivative outputs must be modeled as ordered sections and media placements, not monolithic documents.

## Explicit Non-Goals
- No implementation of generation runtime.
- No implementation of prompt orchestration engine.
- No automated copy output in this planning phase.
