# Methodology Intake (Draft)

Purpose: centralized intake area for human-authored methodology artifacts that will later be translated into system logic.

Status: draft and pending artifact review.

## Expected Inputs
- Written methodology narrative from SEO/content stakeholders
- Spreadsheet/template structure describing blog flow, sections, fields, and checks
- Supporting reference materials and examples

## Folder Structure
- `narrative/`: source narratives, process docs, and written guidance
- `spreadsheets/`: workbook files and exported tab snapshots
- `references/`: external/source references used by the methodology
- `mapping-notes/`: translation notes from human artifacts to system-ready structures

## Intake Rules
- Do not overwrite source artifacts; preserve originals.
- Keep filenames stable and dated when versions change.
- Do not convert draft methodology into final policy/business rules before stakeholder approval.
- Treat all artifacts here as pending until documented in governance decisions.
